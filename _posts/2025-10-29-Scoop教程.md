# Scoop教程

## 安装  scoop-cn（推荐）

此方法会把安装 Scoop 过程中的地址都换成中国可快速访问的地址，并设置好 Scoop，添加本库为 main 库。打开 PowerShell，输入以下命令下载安装：

```powershell
irm https://gh-proxy.org/https://raw.githubusercontent.com/duzyn/scoop-cn/master/install.ps1 | iex
```

## 安装 Scoop

打开 PowerShell：
按下 `Win + X` 快捷键，选择 **Windows PowerShell \(管理员\)** / **终端 \(管理员\)**，首次建议管理员权限运行，规避权限问题。

### 1\. 允许本地运行 PowerShell 脚本（仅首次执行）

管理员终端运行命令，弹窗输入 `Y` 确认：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 2\. 自定义安装路径（安装前执行，可选 D 盘部署）

不想安装至 C 盘用户目录，提前执行下方命令，示例安装路径：`D:\Scoop`

```powershell
# 1. 临时设置Scoop安装目录环境变量
$env:SCOOP='D:\Scoop'
# 2. 永久写入用户环境变量，重启终端依旧生效
[Environment]::SetEnvironmentVariable('SCOOP', $env:SCOOP, 'User')

# 【可选】全局软件安装路径（全局安装-g参数专用）
# $env:SCOOP_GLOBAL='D:\GlobalApps'
# [Environment]::SetEnvironmentVariable('SCOOP_GLOBAL', $env:SCOOP_GLOBAL, 'Machine') # Machine级别必须管理员权限
```

### 3\. 执行官方安装命令

```powershell
irm get.scoop.sh | iex
```

出现 `Scoop was installed successfully!` 即安装成功。
默认路径：`C:\Users\你的用户名\scoop`。

### 安装完成目录结构

```Plain Text
D:\Scoop
├─ apps        # 已安装软件本体
├─ buckets     # 软件源仓库(Bucket)
├─ cache       # 安装包下载缓存
└─ persist     # 软件配置持久化目录
```

## 管理软件仓库

Bucket = Git 软件仓库，内置软件安装清单`Manifest`，默认仅有`main`仓库（命令行工具），需额外添加仓库获取 GUI 软件。

### 添加软件仓库

```powershell
scoop bucket add extras
```

> 注意：添加 Bucket 依赖`Git`，未安装会自动提示安装，可手动前置执行：`scoop install git`
> 克隆仓库慢属于国内 GitHub 网络常态，可配置代理 / 加速工具。

### 删除软件仓库

```powershell
scoop bucket rm main
scoop bucket rm extras
```

## 软件日常管理命令

### 搜索软件

```powershell

scoop search 软件关键词    # 通用格式
scoop search python       # 示例：搜索Python
```

可视化检索：官网 [scoop\.sh](https://scoop.sh/)，取消`main`筛选，可检索全仓库软件。

### 单个软件安装

```powershell
scoop install git        # main仓库
scoop install 7zip       # main仓库
scoop install vscode     # extras仓库
```

### 批量一键安装（空格分隔）

```powershell
scoop install nodejs python mysql nginx putty everything powertoys
```

### 更新操作

```powershell
scoop status                # 检查所有软件可更新状态
scoop update                # 更新Scoop本体 + 所有Bucket仓库清单
scoop update *               # 一键更新全部已装软件
scoop update nodejs         # 更新指定单个软件
```

### 列表与信息查看

```powershell
scoop list                  # 列出本机所有已安装软件
scoop info git              # 查看软件详情：版本、仓库、依赖、历史版本
scoop which git             # 查看命令对应程序真实路径（Linux which 等效）
scoop home vscode           # 浏览器打开软件官网
```

### 卸载与版本回滚

```powershell
scoop uninstall mysql       # 彻底卸载软件（程序+垫片+快捷方式）
scoop reset git@2.30.0.windows.1  # 回退至指定历史版本
```

### 空间清理

```powershell
scoop cleanup *             # 清理所有软件旧版本安装文件，释放空间
scoop cache rm *            # 清空全部下载安装包缓存
```

## 常见问题 FAQ

### 问题 1：添加 Bucket / 安装脚本超时、网络失败

代理配置

```powershell
scoop config proxy http://代理地址:端口  # 设置代理
scoop config proxy                      # 查看当前代理
scoop config proxy none                 # 清空代理
```

1. 辅助方案：Watt Toolkit 加速 GitHub、切换公共 DNS

2. 离线安装：浏览器打开 `get.scoop.sh` 保存为`install.ps1`，本地执行脚本安装。

### 问题 2：命令提示「不是内部或外部命令」

1. 首选方案：**关闭当前终端，重新打开 PowerShell/CMD**（环境变量新会话生效）

2. 修复 PATH：执行 `scoop fix path` 自动修复垫片路径

## 进阶实用技巧

### 1\. 自建私有 Bucket

可自建 Git 仓库编写`Manifest`清单，收录官方仓库没有的私有软件，实现团队软件统一分发。

### 2\. 系统重装一键环境还原脚本

新建`setup-my-env.ps1`，后续重装系统只需执行 Scoop 本体安装，再运行该脚本即可一键部署全套开发软件：

```powershell
# setup-my-env.ps1
# 添加必备仓库
scoop bucket add extras
scoop bucket add java
scoop bucket add nerd-fonts

# 批量安装常用工具
scoop install git python vscode 7zip everything 

# 自动清理旧版本
scoop cleanup *
Write-Host "✅ 开发环境部署完成！"
```

运行脚本：终端进入脚本目录，执行 `.\setup-my-env.ps1`
